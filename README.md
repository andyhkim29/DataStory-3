## Overview

This project asks: **What makes songs popular?**

The analysis uses the Kaggle dataset [900k Spotify](https://www.kaggle.com/datasets/devdope/900k-spotify), which includes Spotify-style audio features, metadata, lyrics, recommendation fields, and a built-in popularity tag. The workflow is designed to:

1. Inspect the available fields and define a defensible popularity target.
2. Use the dataset's direct `Popularity` score as the primary target.
3. Compare the top 10% of tracks by popularity against the remaining full valid dataset.
4. Explain audio-feature drivers and artist/album metadata patterns separately.

## Key Findings

The executed analysis uses 551,435 valid records from the `devdope/900k-spotify`
dataset. Popularity is measured with the dataset's direct `Popularity` field.
Popular songs are the top 10% of tracks by that score, which means
`popularity >= 58` and includes 55,776 tracks.

Evidence charts generated by the notebook:

- [Popularity target distribution](figures/target_distribution.html)
- [Audio feature differences](figures/audio_feature_differences.html)
- [Ranked popularity drivers](figures/ranked_drivers.html)
- [Popular song top-tag Venn diagram](figures/popular_tag_venn.html)

Main findings:

- Popular tracks are shorter on average: 216,644 ms versus 224,397 ms for remaining tracks, a difference of about 7.8 seconds.
- Popular tracks are louder: -7.52 dB versus -8.14 dB.
- Tempo is nearly the same between groups: 120.63 BPM for popular tracks versus 120.50 BPM for remaining tracks.
- Popular tracks are more danceable: 0.634 versus 0.587 after scaling the dataset's percentage-style audio features to 0-1.
- Popular tracks are less acoustic and less instrumental: acousticness averages 0.230 versus 0.261, and instrumentalness averages 0.050 versus 0.074.
- Popularity is more concentrated by albums and artists: the top 10% has an album repetition rate of 0.802 versus 0.695 for the rest, and an artist repetition rate of 0.851 versus 0.751.
- The tag Venn diagram focuses on the three tags that appear most often among popular songs, showing both their individual counts and how often they overlap on the same tracks. Social/activity tags stand out: `Good for Party` appears on 24.5% of popular tracks versus 4.1% of the rest, and `Good for Social Gatherings` appears on 8.4% versus 0.4%.
- Popular tracks are overrepresented in pop, rap/hip-hop/trap, K-pop, pop-country, and pop/R&B genre tags.
- Explicit tracks are more common in the popular segment: 37.4% versus 28.5%.

These findings are associative. They describe which track and metadata patterns
are associated with higher popularity in this Kaggle dataset, not causal drivers
of universal listener behavior.

## Running The Code

1. Create and activate a virtual environment.

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   ```

2. Install dependencies.

   ```bash
   pip install -r requirements.txt
   ```

3. Download the Kaggle dataset and place or symlink the CSV file in `data/raw/`.

   Dataset: https://www.kaggle.com/datasets/devdope/900k-spotify

   KaggleHub command:

   ```python
   import kagglehub
   path = kagglehub.dataset_download("devdope/900k-spotify")
   print("Path to dataset files:", path)
   ```

4. Run the notebook.

   ```bash
   jupyter notebook notebook.ipynb
   ```

5. Optional validation.

   ```bash
   pytest tests/
   jupyter nbconvert --to notebook --execute notebook.ipynb --output notebook.executed.ipynb
   ```

The notebook includes code for discovering and importing data from `data/raw/`. If no data file is present, it prints clear download instructions instead of failing silently.
